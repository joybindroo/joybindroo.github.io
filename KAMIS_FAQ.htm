FAQ - Kala Azar Management Information System (KAMIS)

Last update: Jun 2020

 

1.       How to visit KAMIS portal?

·         Make sure your computer has working Internet connection.

·         Open web browser window/tab. e.g. - Google Chrome, Mozilla Firefox, Microsoft Edge.

·         Type URL of KAMIS  https://www.ka-mis.org and hit enter.

·         KAMIS login page will be loaded and displayed

 

2.       Cannot access KAMIS portal.

·         Make sure internet is connected.

·         If internet is working and still unable to login, then try clearing cache of your web-browser. If you dont know how to clear web browser cache, then you can read more and follow steps mentioned on these links 

-          Clear cache in Google Chrome

-          Clear cache in Firefox

-          Clear cache in Microsoft Edge

·         Contact admin if problem persists.

 

3.       Downloading KAMIS mobile app for Android phones.

·         Open Google Play Store App on your Android mobile device.

·         Type KAMIS in the search box and search for the app.

·         You can also find link to download app on login page of KAMIS.

·         This is direct link to download KAMIS Android app.

4.       Unable to login in portal.

·         Check your login credentials.

·         Type correct username and password.

·         Contact admin if still unable to login.

 

5.       My account is locked.

·         If wrong password is entered multiple times, then login account is deactivated by system for security.

·         You need to contact admin to re-activate your login.

6.       Problem in mobile app login.

·         Check internet connection, internet connection is required for successfully logging in to app.

·         See that you are using right user level credentials.

·         Carefully enter the correct username and password.

·         Contact admin if still failing to login.

 

7.       Can we use mobile app without internet?

·         Yes, mobile app can be used without Internet.

·         Data can be punched in App without Internet connectivity.

·         Internet is required at the time of logging in with authorized credentials, synchronization, and at the time of submitting punched data to server.

 

8.       Problem in submitting data in mobile app of KAMIS.

·         Internet connectivity is required for submitting data from app.

·         Check that you have filled all mandatory fields of data entry

·         Check that you have completed all the required sections of data entry form.

 

9.       How to know about cases pending for treatment or missing treatment information in KAMIS?

·         No. of cases whose treatment details havent been entered are notified in the system.

·         Clicking on the Treatment page link opens the list of cases without treatment information.

·         Facility level users can update treatment details directly from this page by using the Add Treatment link

·         User can also add treatment information by opening the patient record in line list section and then adding the treatment information in the case record.

 

10.   What to do if resident patient couldnt be traced on follow-up?

·         Open patient records from line list page and add follow-up data to case record - selecting Untraceable in the question Whether found alive?

 

11.   What to do if non-resident patient couldnt be verified?

·         Open non-resident patient record from To Verify page and then Reject that case mentioning the relevant information and reasons for rejecting that case.

 

12.   How do I know about the non-resident patients entered by my facility, who have been rejected by the corresponding resident block facility?

·         Open Rejected page available in the non-resident section.

·         List of rejected non-resident cases will be shown.

·         Necessary corrections in the block/district name (if any required) can be made to the rejected records from this page, so that such case records get notified to correct resident facility.

 

13.   What are the important points to keep in mind for accepting non-resident cases as resident patient of block?

·         Block worker need to physically verify that patient is a resident of the block.

·         If such case couldnt be traced in block, or if he/she is not a resident of the block, then reject the case mentioning relevant details.

·         If case gets verified on ground then accept the patient as a resident  From the line list register find out the next available serial no. in the month, then do listing of this additional case with that serial no. by accepting it in online KAMIS and  then physically noting down details of this case in the resident register.

·         If case gets verified and resident entry for same case is existing in register (this could happen when patient gets diagnosed in facility and is referred to other facility for treatment), then block user need to link notified non-resident entry with already existing resident entry of the case. Linking helps check duplication of patient records.

 

 

14.   Problem in linking non-resident case.

·         This happens because of a system check which ensures that module is not used in wrong manner.

·         If non-resident record is diagnosed after more than one year of diagnosis date of resident record, then linking of two records is not allowed.

 

15.   What to do when non-resident case accepted with wrong serial no. in online KAMIS?

·         In this situation, district level user can delete the accepted entry from resident line list section.

·         This will revert the acceptance step and non-resident patient record will again start showing up for verification/acceptance.

·         Now, accept the non-resident case with correct serial no. again.

 

16.   Facing issue while submitting form in online KAMIS.

·         Check that system has internet connectivity.

·         Check that you have filled all mandatory fields in the data entry form.

·         Check that you have completed all the required sections of data entry form.

 

17.   Update treatment information.

·         Health facilities can update treatment details of a KA case using their block level login.

·         Go to Resident/Non-Resident Line List page

·             Search and select the patient

·             Add treatment details

·             Save record.

 

18.   Update follow-up data.

·         Health facilities and outreach workers engaged in active surveillance can update follow-up details of KA cases using their block level login.

·         Go to Resident/Non-Resident Line List page
 Search and select the patient
 Add follow-up details
 Save record.

19.   Wrong/duplicate patient records.

·         Duplicate or wrong records are saved due to data entry errors.

·         District level user can delete resident or non-resident records from KAMIS database for correcting the line listing.

 

20.   How to update death details of KA case?

·         Death of a case can be captured in follow-up section.

·             Go to line list page

·             Search and select the case

·             Open patient detail page

·             Add Follow-up

·             Select 'Dead' option in the question 'Whether patient is alive or dead?

·             Enter other relevant details

·             Save record.

 

21.   How to add payment information?

·         Go to line list page

·             Search and open patient details page

·             Go to Payments Section

·             Add details of wage loss compensation paid by state and center, also enter details of ASHA incentive paid (if applicable).

 

 

22.   Export patient line list

·             Go to line list page

·             Use search options to select required patient list

·             Click on Export button to download data

 

23.   Downloading complete datasets.

·         District, state and national level users can also download complete datasets of various information sections like diagnosis, treatment, follow-up etc.

·             Go to download page

·             Apply filters if required

·             Click on the button of section you wish to download data.

 

24.   View dashboard/statistics

·         Go to statistics page

·             Click on the dashboard link you wish to view.

 

25.   Generate inventory report

·         Inventory status reported by block PHCs, District Hospitals and other Medical Institutions can be accessed through facility level inventory reports.

·         Inventory status at district and state store is available in district store inventory reports and state store inventory store page respectively.

26.   How to view status of payments?

·             Go to payments report

·             Select filters for required report

·             Click on search to generate report on webpage

·             Click on download button to download report data.

27.   Not receiving email notifications.

·         Contact admin to make sure your active email address is registered in KAMIS by admin.

·             Enter old password

·             Enter new password

·             Confirm new password

29.   How to access list of suspects?

·         Block level ACD users can enter suspect details and later access them.

·         Suspects who have been visited and shown in Visited page.

·         Suspects who need to be visited are shown in Pending Visit page.

 

30.   How to view data submitted through IRS module of mobile application?

·         Data can be viewed in online portal by logging in using same block level IRS credentials.

·         District level users can also view submitted IRS concurrent monitoring data.

·         State as well as district level users can also view dashboards for IRS indicators reported from submitted data.
